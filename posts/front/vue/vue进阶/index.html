Vue进阶#

Vue路由#

1.初始路由#

**路由**：表示一种对应关系，某些事物的一一对应的关系

**后端路由**：后端路由是由服务器端进行实现，并完成资源的分发（url与资源/数据的对应关系）

**前端路由**：前端路由是依靠hash值（锚点链接）的变化进行实现页面内容的切换（锚点与组件之间的关系）

2.vue-router#

  1. vue-router工作原理

    1. 它是一个Vue.js官方提供的路由管理器，是一个功能更加强大的前端路由器，可以实现前端界面内容切换

    2. SPA：Single Page Application，单页面应用，整个网站只有一个页面

    3. SPA推荐使用vue-router，实现界面内容的切换

    4. vue-router提供两种模式

      1. hash模式：hash又叫锚点：比如#/login，根据hash改变，来实现界面切换

      2. history模式：history模式不会出现#号比较美观，利用history.pushState()来完成URL的跳转而且无须重写加载页面

        • 示例代码

          const router = new VueRouter({
    mode: 'history',
    router: [...]
})

        • vue-router默认为hash模式，需要通过mode来改变模式

  2. vue-router基本使用

    1. 引入vue-router.js

    2. 定义路由链接和路由视图

      代码示例：

      <div id="app">
    <router-link to="/login" tag="span">前往登录</router-link>
    <router-view></router-view>
</div>

      router-link是路由链接，to代表链接地址，tag代表渲染的标签

      router-view是路由视图，用于显示路由链接对应的组件

    3. 定义路由匹配规则

      代码示例

      var login = {
    template: "<h1>登录组件</h1>"
}
const router = new VueRouter({
    routes: [
        {
            path: '/login',
            component: login
        }
    ]
});

      在实例化VueRouter的时候传入routes

      通过routes定义路由匹配规则，可以定义多个

      匹配规则是一个对象，key是路由链接path，value是组件component

    4. 注册路由

      代码示例

      var vm = new Vue({
    el: '#app',
    router: router
});

      在实例化Vue的时候，通过选项router来注册路由

  3. 路由对象属性

    1. this.$router表示全局路由器对象，就是通过router选项注册的VueRouter路由管理器对象

    2. this.$route代表当前路由对象，而已获取当前路由相关信息

    3. 路由对象属性

      属性名 类型 说明
      $route.path String 对应当前路由的路径
      $route.query Object 一个{key:value}对象，表示URL查询参数
      $route.params Object 一个{key:value}对象，路由跳转携带参数
      $route.hash String 在history模式下获取当前路由hash值（带#），如果没有hash值，则为空字符串
      $route.fullPath String 完成解析后的URL，包含查询参数和hash的完整路径
      $route.name String 当前路由的名称
      $route.matched Array 路由记录，当前路由下路由声明的所有信息，从父路由（如果有）到当前路由为止
      $route.redirectedFrom String 如果存在重定向，即为重定向来源的路由

4.动态路由#

  1. 什么是动态路由

    可以匹配带有不同参数的url的路由规则就是动态路由

  2. query方式传参

    1. 通过？传参

    2. 传参

      1. 示例

        <div id="app">
    <router-link to="/user?id=1&name=Rex">登录</router-link>
    <router-view></router-view>
</div>
<template id="user">
    <h1>
        id: {{this.$route.query.id}} name: {{this.$route.query.name}}
    </h1>
</template>

      2. 传参语法：?参数名1=参数值1&参数名2=参数值2

    3. 定义路由

      const router = new VueRouter({
    routes: [{
        path: '/user',
        component: user
    }]
})

    4. 获取参数

      {{this.$route.query.参数名}}

  3. params方式传参

    1. 通过url本身传递

    2. 传参

      1. 示例代码

        <div id="app">
    <router-link to="/user/10/admin">登录</router-link>
    <router-view></router-view>
</div>
<template id="user">
    <h1>
        id:{{this.$route.params.id}} name:{{this.$route.params.name}}
    </h1>
</template>

      2. 传参语法：/参数值1/参数值2，这里只需要传递值即可

    3. 定义动态路由

      1. 实例代码

        const router = new VueRouter({
    routes: [
        { path: "/user/:id/:name", component: user }
    ]
})

      2. 动态路由语法，:参数名，以冒号开头就代表动态匹配

      3. :id就会匹配10，:name就会匹配admin

    4. 获取参数

      {{$route.params.参数名}}

5. 嵌套路由#

  1. 什么是嵌套路由

    1. 界面中如果存在嵌套组件,那么就需要组件对应的路由也是嵌套路由

    2. 定义嵌套路由:

      1. 通过在路由对象中定义children属性,定义嵌套的子路由

      2. 示例代码

        var router = new VueRouter({
    routes:[{
        path: '/home',
        component: home,
        children:[
            {path: 'login', component: login},
            {path: 'register', component: register}
        ]
    }]
})

    3. 使用嵌套路由

      1. 示例代码:

        <router-link to="/父路由的地址/要去的子路由"></router-link>

      2. 子路由的url是:父path/子path

6. 命名路由#

  1. 如果路由的path比较长,使用的时候比较麻烦,这时候可以给路由起个名字,用的时候通过名字使用

  2. 定义命名路由

    1. 示例代码

      ​

      const router = new VueRouter({
    routes: [
        { path: '/user/:id', name: 'user', component: user }
    ]
})

    2. 在定义路由对象时,通过name来给路由起名字

  3. 使用命名路由

    1. 示例代码

      <router-link :to="{name:'user', params: {id:123}}">登录</router-link>

    2. 使用属性绑定,是因为to的值是一个js对象

7. 命名视图#

  1. 给视图起名字,就是命名视图

  2. 在开发中想同时显示多个同级视图,这时候就需要使用命名视图

  3. 定义命名视图

    1. 示例代码

      <div id="app">
    <router-view></router-view>
    <div class="container">
        <router-view name="left"></router-view>
        <router-view name="main"></router-view>
    </div>
</div>

    2. 通过name属性给router-view添加名字

  4. 路由规则使用命名视图

    1. 示例代码

      var header = {
    template: '<h1 class="header">header头部区域</h1>'

    2. 路由对象中通过components来使用多个视图

8. 编程式导航#

  1. router.push()

    1. push:推,push方法可以实现url跳转

    2. 语法:

      1. //先获取router实例
var router = new VueRouter()
//字符串形式
router.push('user')
//对象形式
router.push({path:'/login?url='+this.$route.path})
//命名路由
router.push({name:'user', params:{userId:123}})
//带查询参数/user?id=1
route.push({path:'user',queyr:{id:'1'}})

      2. push可以传递一个字符串路径,也可以是对象

      3. 对象中可以是路径path,也可以是路由的名字

    3. 注意

      1. 在参数对象中,如果提供了path,params会被忽略,为了传参数,需要提供路由的name或者手写带有参数的路径

      2. const userId = '123'
router.push({name:'user', params:{userId}}) // /user/123
router.push({path:'/user/${userId}'})		// /user/123
//这里的params不生效
router.push({path:'/user', params:{userId}})// /user

  2. router.replace()

    1. replace:替换,可以替换当前界面的url

    2. 语法

      //编程式
router.replace({path:'user'})
//声明式
<router-link :to="{path:'user'} replace"></router-link>

    3. 与push的区别:push有history记录,replace没有

  3. router.go()

    1. go:去,可以实现前进后退
    2. go()方法,接受正数或负数,表示前进或后退
    3. 后退一个界面:$router.go(-1)
    4. 前进一个界面:$router.go(1)

Vuex状态管理#

初始Vuex#

  • Vuex是一套组件状态（数据）管理维护工具

  • 使得Vue组件的状态更加容易维护

  • 创建Vuex的store示例

    • const store = new Vuex.Store({
	state:{},
    mutations:{}
})

    • 使用Vuex就是使用store来对状态进行管理

    • Vuex通过store(存储来存储数据)，并且管理数据

Vuex的下载和安装#

  1. script引入文件

    1. 下载Vuex

    2. 引入

      <script src="../js/vuex.js"></script>

    3. 使用

      1. 创建store

        const store = new Vuex.Store({
    state: {
        name: 'vuex.js直接引用'
    }
})

        实例化Store的时候，可以传入state选项

        state选项：状态选项，数据选项，用于定义要存储的数据

      2. 挂载store

        var vm = new Vue({
    el: '#app',
    store
})

      3. 使用state数据

        <div id="app">
    <p>
        {{this.$store.state.name}}
    </p>
</div>

        获取state数据是一层一层的获取的

        this中有store选项，所以：this.$store

        store中有state，所以this.$store.state

        state中有name，所以this.$store.state.name

      4. 使用计算属性

        1. 上边的方式，使用state数据，太麻烦了，每次都要this.$store.state.name

        2. 我们可以在计算属性中将state数据返回，相当于将state数据变为计算属性，这样在使用的时候就简单很多

        3. 定义计算属性

          var vm = new Vue({
    el: '#app',
    store,
    computed: {
        name() {
            return this.$store.state.name;
        }
    }
})

        4. 使用

          <div id="app">
    <p>
        {{name}}
    </p>
</div>

      5. mapState辅助函数

        1. 当一个组件需要获取多个状态时，将这些状态都声明为计算属性有些麻烦，这时候可以使用mapState辅助函数来生成计算属性

        2. 使用mapState

          var vm = new Vue({
    el: '#app',
    store,
    computed:
    Vuex.mapState({
        //key:计算属性的名字
        //value:函数，将state数据返回
        name: state => state.name
    })
})
          • mapState是Vuex提供的一个方法
          • mapState，是状态映射，理解为将state数据映射（转换）为计算属性
          • mapState接受一个对象，对象中可以映射多个数据
          • 对象中的key是计算属性的名字，value是一个箭头函数，箭头函数有一个state参数然后直接将state的数据返回即可

        3. 使用数据

          <div id="app">
    <p>
        {{name}}
    </p>
</div>

Vuex状态管理模式#

  1. 单项数据流机制：在Vue中，组件的状态变化是通过Vue单向数据流的设计理念实现的。

  2. 单项数据流组成部分主要包括

    1. State：驱动应用的数据源
    2. View：以声明方式将state映射到视图
    3. Actions：响应在View上的用户输入导致状态变化

  3. 代码:

    new Vue({
    //State
    data(){
        return {count: 0}
    },
    //View
    template: '<div>{{ count }}</div>',
    //Actions
    methods:{
        increment(){
            this.count++;
        }
    }
})

Vuex的工作流程关系#

actions、mutations和state的关系

  1. State数据render渲染到vue组件
  2. 用户在Vue组件中点击按钮，触发逻辑Actions
  3. Actions中通过commit触发修改数据逻辑Mutations
  4. Mutations修改的state，重新渲染到Vue组件

Vuex配置选项#

1.mutations#

  1. mutations是一个对象，可以定义多个方法

  2. mutations方法用来操作state数据（翻译：变化，理解为数据变化）

  3. mutations方法接受state对象作为参数

  4. 在组件中通过store.commit(“mutations方法名”)来触发mutations方法执行

  5. 代码：

    const store = new Vuex.Store({
    state: {
        param: ""
    },
    mutations: {
        reciver(state, param) {
            this.state.param = param;
            console.log(param);
        }
    }
})

    new Vue({
    el: "#app",
    store: store,
    methods: {
        param() {
            this.$store.commit({ type: "reciver", name: "我是传递的数据" })
        }
    },
})

2.actions#

  1. actions类似于mutations，不同之处在于actions是异步执行的（翻译：动作，理解为执行异步动作）

  2. actions函数接受store对象，用于完成mutations提交

  3. store的dispatch方法触发actions函数

  4. 代码

    1. const store = new Vuex.Store({
    state: {
        count: 0
    },
    //mutations：专门用来修改state的数据
    mutations: {
        increment(state) {
            state.count++;
        }
    },
    actions: {
        add(context) {
            setTimeout(() => {
                context.commit("increment")
            }, 1000);
        }
    }
})

      1. context即为store实例

      2. 简写

        actions:{
	add({commit}){
       commit("increment")		
	}	
}

    2. var vm = new Vue({
    el: '#app',
    store,
    methods: {
        calc() {
            this.$store.dispatch("add");
        }
    },
});

      store的dispatch方法，用于触发actions方法

3.getters#

  1. getters，类似于Vue实例的computed计算属性（翻译：获取者，获取计算之后的数据）

  2. 定义

    const store = new Vuex.Store({
    state: {
        todos: [{
            id: 1,
            text: '列表1'
        }, {
            id: 2,
            text: '列表2'
        }],
        id: 0
    },
    mutations: {
        searchM(state, id) {
            state.id = id;
        }
    },
    getters: {
        searchG: state => {
            return state.todos.filter(
                todo => (todo.id == state.id)
            )
        }
    }
})

    getters是计算属性，计算属性其实是方法，只不过使用的时候是属性

  3. 使用

    <div id="app">
    <h2>列表查询</h2>
    <input type="text" v-model="id">
    <button @click="search">搜索</button>
    <p>搜索结果：{{this.$store.getters.searchG}}</p>
    <ul>
        <li v-for="item in this.$store.state.todos">{{item}}</li>
    </ul>
</div>

4.modules#

  1. modules模块就是为了应对复杂的业务逻辑，将数据与操作数据的逻辑分模块处理

  2. modules的配置选项：

    1. 与store数据参考中的参数是相同的

    2. key:{		//key表示模块名称
    state,	//初始数据
    mutations,	//状态提交，同步
	actions,	//状态分发，异步
    getters,	//计算属性
    modules		//模块
}

    3. 代码

      const moduleA = {
    state: {
        nameA: 'A'
    }
}
const moduleB = {
    state: {
        nameB: 'B'
    }
}
const store = new Vuex.Store({
    modules: {
        a: moduleA,
        b: moduleB
    }
});
console.log(store.state.a);
console.log(store.state.b);

5.plugins#

  1. 用于给Vuex添加插件，插件本身为函数

  2. 函数接受参数store对象作为参数

  3. 代码

    const myPlugin = store => {
    store.subscribe((mutation, state) => {
        // 每次mutation提交后调用，mutation格式{type, paylooad}
        console.log(mutation.type, mutation.payload);
    })
}
const store = new Vuex.Store({
    mutations: {
        do(state) {
            console.log(state);
        }
    },
    plugins: [myPlugin]
});
var vm = new Vue({
    el: '#app',
    store,
});
store.commit('do', 'plugin')

    插件函数中通过store实例对象的subscribe函数可以用来处理mutation

    subscribe函数接受参数为mutation和state

  4. 作用：一般在项目中我们会使用plugin初始化一些数据

6.devtools#

  1. 用来设置是否在devtools调试工具中应用Vuex
  2. 默认值为true，表示在启用，设为false表示停止使用

Vuex中的API#

模块注册#

  1. store.registerModule()用于动态创建模块的接口

  2. 与modules选项作用类似，但是registerModule()更加灵活

  3. 代码

    1. const store = new Vuex.Store({
    state: {},
});
store.registerModule('myModule', {
    state: {
        name: "我是注册模块的数据"
    }
})

      modules选项只能在实例化store的时候，注册模块

      registerMudule()方法只要是能够获取到store对象的地方都可以注册

状态替换#

  1. store.replaceState()方法用于实现状态替换

  2. 该方法接受新的state对象

  3. 代码

    const store = new Vuex.Store({
    state: {
        name: '我是初始值数据'
    }
});
store.replaceState({ name: "替换后的新数据" })
var vm = new Vue({
    el: "#app",
    store,
})

    <div id="app">
    <p>{{this.$store.state.name}}</p>
</div>
