Vue基础#

初始Vue#

前端技术的发展#

  1. HTML：用于编写网页的结构

  2. CSS：用于美化网页

  3. JS：用于处理页面的交互效果

  4. JQuery：JS的升级版，代码简洁，更高效的开发项目

  5. Vue：采用虚拟DOM技术，代码更加简洁

    虚拟DOM技术

    • 和真实DOM结构类似的JS对象
    • 可以提高浏览器的渲染速度

什么是Vue#

  1. 一条基于构建用户界面的渐进式框架

    • 渐进式理解：Vue提供的一系列技术，可以渐渐的，一步一步的使用这个框架
    • 框架理解：解决一类问题的解决方案，理解为工具

  2. Vue的数据绑定基于MVVM设计思想

    MVVM：Model，View，ViewModel

    可以将模型数据与视图连接起来

    • 模型数据变化会影响视图
    • 视图数据变化会影响模型

Vue的优势#

  • 轻量级：Vue简单、直接，所以Vue使用起来更加友好
  • 数据绑定：数据驱动视图，视图也可以驱动数据
  • 指令：指令绑定在元素上时，指令会给绑定的元素添加一些特殊的行为
  • 插件：插件用于对Vue框架功能进行扩展

Vue、Angular和React的区别：#

  • Angular相对Vue学习成本高，更复杂，性能稍差
  • React采用了特殊的JSX语法，Vue中推崇编写以*.vue后缀命名的文件格式

Vue开发基础上#

1.Vue实例#

  1. 说明

    1. Vue实例：通过new关键字实例化Vue({})构造函数

    2. 实例化Vue时的参数内的属性，我们称之为选项：

选项 说明
data Vue实例数据对象
methods 定义Vue实例中的方法
component 定义子组件
computed 计算属性
filters 过滤器
el 唯一根元素
watch 监听数据变化
  1. el唯一根标签：在创建Vue实例时，el表示唯一根标签，通过指定class或id选择器可用来将页面结构与Vue实例对象vm进行绑定。
    1. el：element，元素
    2. el选项表示：当前Vue实例绑定的页面怨怒是
    3. 绑定的这个元素，作为Vue实例在界面上的元素
    4. el选项的值：“id或类选择器”
    5. 注意：Vue的数据只能在el绑定的元素内部使用
  2. data初始数据：Vue实例的数据对象为data，Vue会将data的属性转换为getter、setter，从而让data的属性能够响应数据变化。
    1. data：数据
    2. data选项表示：当前Vue实例可以使用的数据
    3. data选项的值：对象，对象中可以定义多个数据
    4. js中获取data数据：vm.$data.name、vm.name
  3. methods定义方法：
    1. method：方法，为啥加s，因为可以有多个方法
    2. methods选项表示：当前Vue实例可以用到的方法
    3. methods选项的值：对象，对象中定义方法
  4. computed计算属性：计算属性结果会被缓存起来，当依赖的响应式属性发生变化时，才会重新计算，返回最终结果。
    1. computed：计算
    2. computed选项表示：当前Vue实例可以用到的计算属性
    3. computed选项的值：对象，对象中定义方法
    4. 应用场景：当有些数据需要随着其他数据变动而变动时，就需要使用computed计算数学；一般处理计算逻辑
    5. 注意：计算属性，定义时是一个方法，但是当属性使用（所以才叫计算属性）；计算属性需要将结果放回；计算属性具有缓存，这就意味着只要计算属性的依赖没有进行相应的数据更新，那么computed会直接从缓存中获取值，多次访问都会返回之前的计算结果
  5. watch状态监听：用来检测Vue实例中的数据变动
    1. watch：观察、检测
    2. watch选项表示：检测data选项中的数据变化
    3. watch选项的值：对象，对象中定义方法
    4. 应用场景：要在数据变化同时进行异步操作或者是比较大的开销时使用
    5. 注意：watch是检测data的数据变化，watch的方法必须与data的属性名一致
  6. filter过滤器：对数据进行格式化，比如字符串字母变大写，日期格式化等。
    1. filter：过滤器
    2. filters选项表示：对data选项中的数据进行格式化
    3. filters选项的值：对象，对象中定义方法
    4. 应用场景：对数据格式化（字符串中首字母大写，字符串大小写转换、日期格式化）
    5. 注意：过滤器函数，必须定义一个参数，此参数就是使用过滤器的数据，将格式化的结果返回
    6. 使用过滤器：插值表达式使用：{{data属性名 | 过滤器名称}}或属性绑定使用 v-bind="{{data属性名 | 过滤器名称}}"

2.Vue数据绑定#

  1. 绑定样式：内联样式和绑定样式

    1. 绑定行内样式style
      1. 绑定对象语法：v-bind:style="{backgroundColor:pink, width:width, height:height}
      2. 绑定数组语法：v-bind:style="[myDiv, xxx]"，数组中是一个一个的样式对象，如果只绑定一个对象中括号可以省略
      3. 上述语法中：pink，width，myDiv均为data数据
      4. 为什么要进行样式绑定：绑定之后可以通过操作data数据来改变样式，理解为绑定样式就是为了操作样式
    2. 绑定类样式class
      1. 绑定对象语法：v-bind:class="{box}",data中需要有一个属性：box:“box”,这里的大括号可以省略
      2. 绑定数组语法：v-bind:class="[box1,box2]"

  2. 内置指令

    • 常用内置指令：

      路径 说明
      v-model 双向数据绑定
      v-on 监听事件
      v-bind 单向事件绑定
      v-text 插入文本内容
      v-html 插入包含html的内容
      v-for 列表渲染
      v-if 条件渲染
      v-show 显示隐藏

    • 详解

      1. v-model
        1. v-model主要实现数据双向绑定，通常用在表单元素上，例如input、textarea、select等
        2. 双向数据绑定：数据改变影响界面，界面改变影响数据
      2. v-text指令：
        1. v-text是在DOM元素内部插入文本内容，类似于之前的innerText，理解：v-text是将元素内部文本与data属性进行数据绑定
        2. 这个实现的是单项绑定：数据改变影响界面，处理v-model其他的数据绑定都是单向的
      3. v-html指令：
        1. v-html是在DOM元素内部插入HTML内容，类似于之前的innerHTML，理解：v-text是将元素内部HTML与data属性进行单向数据绑定
      4. v-bind指令：
        1. v-bind可以实现属性单向数据绑定
        2. 目的是为了操作DOM元素的属性
      5. v-on指令：
        1. v-on是事件监听指令，负责给DOM元素绑定事件，配合事件类型使用
        2. 语法：v-on:事件名 = “事件处理程序”
      6. v-for指令：
        1. v-for可以实现页面列表渲染，常用来循环数组
        2. 语法：v-for = “(value, key) in xxx”;
      7. v-if和v-show指令：
        1. 用来控制元素显示或隐藏，属性为布尔值
        2. 语法：v-if=“isShow”，isShow是data中的一个布尔类型数据
        3. 区别：v-if控制元素是否渲染，v-show控制元素是否显示，操作display属性

3.Vue事件#

  1. 事件监听：在Vue中可使用内置指令v-on监听DOM事件，并在触发时运行一些Javascript代码，或绑定事件处理方法。

  2. 语法：v-on:事件名 = “事件函数”；v-on:事件名 = “js代码”

  3. 按键修饰符

    1. 监听哪个键盘/抬起
    2. v-on:keyup.案件修饰符 = ‘xxx’
    3. 案件修饰符其实就是按键名，代表监听哪个按键

  4. 事件修饰符

    常用事件修饰符：

    修饰符 说明
    .stop 阻止事件冒泡（如:阻止点击子元素触发父元素的点击事件）
    .prevent 阻止默认事件行为（a标签单击的默认跳转）
    .capture 事件捕获（如：先执行父元素的点击事件再执行子元素的点击事件）
    .self 将事件绑定到自身，只有自身才能触发（不能冒泡触发）
    .once 事件只触发一次

4.Vue组件#

  1. 什么是组件

    1. 组件是构成页面中独立结构单元，组件主要以页面结构形式存在，不同组件也具有基本交互功能

    2. 组件理解为自定义标签即可，可以是一个标签组成，也可以是多个标签组成

    3. 组件的定义：Vue.component(id, [definition]);

      参数id：组件名称

      参数[definition]：组件配置对象，包括data、template

  2. 局部注册组件

    1. 局部组件，通过components选项来定义
    2. 全局组件与局部组件的区别
      1. 全局组件再项目中的任何界面都可以使用
      2. 局部组件只能在当前界面中使用

  3. template模板

    1. Vue提供了template标签，可以编写组件的结构
    2. 优势
      1. 改善开发体验：之前的组件模板都是直接以字符串形式书写，这种方式很容易出错
      2. 有代码提示，代码高亮，提高开发效率

  4. 组件之间数据传递：在Vue开发中，我们会使用大量的组件，所以会存在组件嵌套的情况，那么子组件之间经常需要传递数据

    1. props传值
      1. props：properties：意为通过属性传智
      2. 这种方式是父组件向子组件传值
        1. my-parent是父组件
        2. template中的标签就是子组件
        3. 父组件在使用的时候，通过属性，给子组件div传递值
    2. $emit传值
      1. emit：发射，理解为触发
      2. Vue实例中提供了一个$emit方法，专门用于触发自定义事件的执行

  5. 组件切换

    1. 通过v-if和v-else来控制是否显示组件

5.Vue生命周期#

  1. 钩子函数：钩子函数用来描述Vue实例从创建到销毁的整个生命周期
    1. Vue生命周期：Vue从创建到销毁的整个过程
    2. 在生命周期过程中，会执行很多回调函数，这些回调函数叫钩子函数
  2. 实例创建
    1. beforeCreate:创建实例对象之前执行，【此时数据不能使用】
    2. created:建实例对象之后执行，【此时数据可以使用】
  3. 页面挂载
    1. beforeMount:页面挂载成功之前执行，【无法展示数据】
    2. Mounted: 页面挂载成功之后执行，【可以展示数据】
  4. 数据更新
    1. beforeUpdate:组件更新之前执行，【数据更新之前】
    2. update:组件更新之后执行，【数据更新之后】
  5. 实例销毁
    1. beforeDestroy:实例销毁之前执行，【可以操作DOM】
    2. destroyed:实例销毁之后执行，【无法操作DOM】

    data() {
        return {
            message: 'mixin的数据'

    }
}
var vm = new Vue({
    data: {
        message: 'Vue的数据'
    },
    mixins: [myMixin],
    created() {
        console.log(this.message);
    }
})

      render(createElement) {
    return createElement('p', {
        style: {
            color: 'red',
            fontSize: '16px',
            backgroundColor: '#eee'
        }
    }, this.$slots.default);
},

    3. 说明

      1. render是一个函数，他作为一个选项存在，可以在Vue和组件中使用这个选项
      2. render的参数：createElment是一个用于创建元素的函数

  3. createElement

    1. 用于创建元素

    2. 说明

      createElement，createElement()函数返回的并不是一个实际的DOM元素，

      他返回的其实是一个描述节点(createNodeDescription)。

      • 第1个参数可以是一个HTML标签名或组件选项对象
      • 第2个参数是可选的，可以传入一个与模板中属性对应的数据对象。
      • 第3个参数是由createElement()构建而成的子级虚拟节点，也可以使用字符串来生成文本虚拟节点。
