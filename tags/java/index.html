Emos项目学习记录

Emos项目学习记录 学习点： 加密算法 这里采用的是IBM开发的AES算法。这种对称加密算法在加密和解密数据的时候，使用相同的密钥。也就是说密钥既可以用来加密数据，也可以解密数据。关于AES算法的优点我这里不展开说明了，大家可以自己百度查阅资料。MySQL数据库提供了内置的DES加密和解密的函数，我们只需要调用即可。加密的函数叫做AES_ENCRPT()，解密的函数叫做AES_DECRPT() ​ 数据库：登录sql，若查询记录结果只有一条，可以加limit 1加快查询个数 <select id="login" parameterType="com.example.emos.api.emos.model.form.LoginForm" resultType="Integer"> SELECT id FROM tb_user WHERE username = #{username} AND password = HEX(AES_ENCRYPT(#{password}, #{username})) LIMIT 1; </select> 数据库：数据库查询操作的返回值用封装类，防止结果为null值不能赋给一般数据类型；若是增删改返回结果肯定为数字且不为null，表示影响的记录个数，可以用一般数据类型 数据库：要不要用MySQL方言？，可能会存在可移植性差 后端：不能随意删除记录，需要查询有没有相关的引用 数据库：写sql的时候需要注意状态字段(status) 后端：分页查询的时候限制分页的start(1, +∞)，size(10, 50) 数据库：灵活使用数据库中的json字段 后端工作流中采用了QuartZ定时器引起、Activeti引擎 业务：审批会议前需注意：1.结束时间必须大于开始时间 2.会议开始时间不能早于当前时间 前端：可用day.js转换日期格式 数据库：数据类型的数据范围、 日期格式： datetime: 8字节 timestamp：4字节 表示范围有限：~2038年就满了...
